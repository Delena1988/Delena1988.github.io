随访2021年规划

发表于 2021-01-06 更新于 2023-03-01 分类于 Jh
本文字数： 832 阅读时长 ≈ 3 分钟

经营目标

智随访： 合同额 3600 回款 2400 预算 1314

家人要和经营目标匹配，加在关键位置上

技术架构优化和产品架构优化方案？

  • 技术架构
    • 技术老旧（Sturts2、springMVC、jsp、jQuery、LayUI）– 开发效率、前端招人
    • 数据库Mysql 5.6（版本老，漏洞多）
    • 前后端未分离，前段开发测试依赖后台
    • 各个模块相互耦合，存在循环依赖
  • 优化方案（优势、弊端、难点、需要的资源）
    • 前后端分离
      • 优势：1、提升开发效率 2、有利于吸引人才
      • 弊端：
      • 难点：1、改造工作量大
      • 需要的资源：1、前端大牛（5年以上，有重构经验的）
    • 拆分成微服务
      • 优势：1、解耦 2、方便扩展 3、容错性好
      • 弊端：1、运维难度提升 2、复杂性提升 3、宣传
      • 难点：1、业务模块划分 建议科技板块专门小组维护公用模块组件
  • 优化需解决问题
    • 老医院兼容（升级）
    • 老医院个性化需求、院内对接需要有解决方案
    • 系统庞大，如何平缓过度
    • 微服务运维成本高，部署实施难度大
    • 微服务资源如何分配（不同医院使用模块不同，业务重心不同）
    • 事物如何控制
    • 模块如何拆分

标准化产品开发进度和标准化实施进度和责任人安排

  • 客服中心随访
  • 病区护士随访
  • 科室专病随访（分支路径自动切换、统计）
  • 单病种数据库（生物样本库已完成，ibd查随访）
  • 满意度
  • 宣教中心
  • 日间手术
  • 投诉表扬（第4套流程开发中）
  • 预约挂号（自治区预约挂号平台）
  • 智问询（待开发）

标准化产品（病区护士随访、客服中心随访、满意度调查、专业版专病、高配版专病）

  1. 病区护士随访
  2. 客服中心随访
  3. 满意度调查
  4. 专业版专病平台，高配版专病（病重管理类）
  5. 诊疗反馈/随访医嘱（医生接诊，下医嘱时制定随访任务）
  6. 面访
  7. 提醒中心（复诊提醒、孕检提醒、用药管理拆分开）
  8. 患者管理

架构

  1. 前后端分离
  2. 后端拆分成微服务

人员画像

  1. 产品2人：1产品总监（对标纪翔），1产品经理
  2. 前端3人：1初级，1中级，1高级（5年以上，有重构经验，技术选型）
  3. 后台3人：2初级，1高级（5年以上，有重构经验）
  4. 测试2人：1初级（1年），1中级（2年以上，会自动化测试以及压力测试）
